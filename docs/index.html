Fusion 360 → URDF · ROS 2 · MJCF · OpenUSD

One export from Fusion 360, every robotics format you need: plain URDF for Isaac Sim, a colcon-buildable ROS 2 Jazzy package, MJCF for MuJoCo, and layered OpenUSD assets via the newton-physics converters — including the Newton-ready flavor with mjc:* physics attributes.

Both USD flavors ship with an Asset Transformer rule profile + runner that restructures them into the official Isaac Sim asset layout (payloads, physics variants, Isaac robot schema) — validated end to end inside Isaac Sim 6.

Quick start

Install the Fusion 360 script

Clone the repo and copy the add-in into Fusion's scripts folder.

# Windows (PowerShell)
git clone https://github.com/johnnynunez/fusion2urdf
Copy-Item ".\addin\URDF_Exporter_Plus\" -Destination "${env:APPDATA}\Autodesk\Autodesk Fusion 360\API\Scripts\" -Recurse

# macOS
cp -r ./addin/URDF_Exporter_Plus "$HOME/Library/Application Support/Autodesk/Autodesk Fusion 360/API/Scripts/"

Export from Fusion 360

Name one component base_link, use Rigid / Revolute / Slider joints with limits, then run URDF_Exporter_Plus from UTILITIES → ADD-INS. It writes <robot>_export/ with the bundle, the URDF and the ROS 2 package.

Build the remaining targets

USD conversion needs pip packages, so it runs on the desktop.

pip install "fusion2urdf[usd] @ git+https://github.com/johnnynunez/fusion2urdf"
fusion2urdf build myrobot_export/ --targets urdf,ros2,mjcf,usd,usd-newton

Load it anywhere

# Isaac Sim — USD opens directly, or full PhysX import:
$ISAAC_SIM_DIR/python.sh out/urdf/isaac_sim_import.py
# Official Isaac Sim asset layout (Isaac Sim ≥ 6.0):
$ISAAC_SIM_DIR/python.sh out/usd/run_asset_transformer.py
# ROS 2:
ros2 launch myrobot_description display.launch.py

Check your export in the browser

Drag your exported out/urdf/ folder into the web viewer: inspect the kinematic tree, move every joint with sliders, and catch wrong axes or origins before touching a simulator. Nothing is uploaded — it all runs locally in your browser.
